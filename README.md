# דשבורד פלייסקייפ — דמו

> ⚠️ **דמו** — כל הנתונים בפרויקט הזה (שמות פרויקטים, מספרים כספיים, עובדים) **בדויים לגמרי**, נוצרו לצורך תיק-עבודות. אין כאן שום קשר לנתונים אמיתיים של שום לקוח או עסק.

זהו עותק-דמו של דשבורד ניהול-סטודיו אמיתי שבניתי ומשתמש בו יום-יומי בעבודה. הדשבורד מתחבר במקור ל-monday.com ומציג תכנון פיננסי, מעקב אבני-דרך בפרויקטים לפי מדיניות תשלום, תחזית עומס-עבודה שבועית, ומעקב עומס-עובדים. בגרסה הזו, שכבת ה-backend מחזירה נתונים מומצאים במקום להתחבר למאנדי האמיתי - כך שאפשר להראות את המערכת בשלמותה בלי לחשוף שום מידע עסקי אמיתי.

## מה כלול

- **באקלוג** - לוח בקרה פיננסי: תכנון מול ביצוע, מנגנון חוב/זיכוי, פרויקטים עם מדיניות-תשלום (אבני-דרך), פרויקטים במעקב בנק-שעות.
- **נפח עבודה צפוי** - תחזית עומס-עבודה שבועית לפי פרויקט, כולל זיהוי משימות חורגות.
- **עומס עובדים** - מעקב עומס חודשי בפועל לפי עובד.
- **ניהול פרויקט** (לשונית נסתרת) - דאבל-קליק על הריבוע הקטן הבלתי-נראה בפינה השמאלית-העליונה של המסך.

## ארכיטקטורה

שני חלקים נפרדים, בדיוק כמו המערכת האמיתית:

- **`web/`** - React 19 + Chakra UI 3 + Vite. אתר רגיל (לא תוסף-דפדפן, בשונה מהגרסה האמיתית שרצה כתוסף כרום) - נפתח עם לינק, בלי התקנה. כל הקוד (קומפוננטות, hooks, לוגיקת חישוב פיננסי) מועתק כמעט-ללא-שינוי מהמערכת האמיתית - הוא לא "יודע" בכלל אם המידע מגיע ממאנדי אמיתי או ממחולל-דמו.
- **`worker/`** - Cloudflare Worker (JavaScript רגיל, בלי TypeScript). מריץ בדיוק את אותה שכבת ניתוב/אימות/מסד-נתונים (D1) כמו המערכת האמיתית - ההבדל היחיד: `fakeData.js` מייצר נתונים מומצאים במקום לפנות ל-API של מאנדי.

### איך הנתונים המומצאים נבנים

`worker/src/fakeData.js` מייצר עולם קבוע (6 פרויקטים, כ-40 משימות, 5 עובדים) שמכסה את כל התרחישים המרכזיים של המערכת: פרויקט כספי רגיל, פרויקט עם מדיניות-תשלום (אבני-דרך מכמה סוגי-טריגר, כולל אחת שממתינה-לאישור), פרויקט במעקב בנק-שעות-בלבד, פרויקט עם משימות פרילאנס-בלבד, משימה שחרגה מהערכת-השעות שלה, ו"משימה יתומה" (סומנה כבוצעה בלי שעות רשומות). כל תאריך בנתונים מחושב **יחסית לרגע האמיתי** (לדוגמה "התחיל לפני 7 חודשים") ולא כתאריך קבוע - כך שהדמו נשאר תקין ומעודכן לצמיתות, גם בעוד שנה, בלי שום תחזוקה.

`worker/smoke-test-fakedata.mjs` הוא בדיקת-תקינות מקומית שמריצה את המחולל ומוודאת שכל תרחיש אכן קיים בפלט - שווה להריץ (`node smoke-test-fakedata.mjs` בתוך `worker/`) אחרי כל שינוי במחולל.

## הרצה מקומית

```
cd web
npm install
npm run dev
```

יפתח בכתובת מקומית (למשל `localhost:5173`). קוד הכניסה: ראו `web/src/lib/api.js`'s `WORKER_URL` להגדרת כתובת ה-Worker (ראו "פריסה" למטה).

## פריסה

### 1. Cloudflare Worker (ידני)

1. יצירת Worker חדש בקלאודפלייר (Workers & Pages → Create → Start from scratch).
2. הדבקת כל קבצי `worker/src/*.js` בעורך המולטי-קובץ (Quick Edit).
3. יצירת מסד-נתונים D1 חדש, חיבורו ל-Worker כ-binding בשם `DB`.
4. הרצת כל 9 קבצי `worker/migrations/000*.sql` בקונסולת ה-SQL של D1, לפי הסדר.
5. הרצת `worker/migrations/seed.sql` (נתוני-הפתיחה הבדויים - מדיניות-תשלום, אירועי-לוז, תאריכי-סיום-משימה) **אחרי** המיגרציות.
6. קביעת secret בשם `ENTRY_CODE` (קוד-כניסה לדשבורד - כל ערך שתבחר, לא רגיש כי אין נתון אמיתי להגן עליו).

### 2. האתר (web/) - אוטומטי דרך Cloudflare Pages

חיבור הרפוזיטורי הזה ל-Cloudflare Pages ("Connect to Git") - כל `git push` יפרסם גרסה מעודכנת אוטומטית. אחרי הפריסה הראשונה של ה-Worker, עדכנו את `WORKER_URL` ב-`web/src/lib/api.js` לכתובת ה-Worker שנוצרה, ו-push.

## איך זה שונה מהמערכת האמיתית

- **בלי חיבור למאנדי בכלל** - `fakeData.js` מחליף את כל שכבת ה-GraphQL.
- **בלי נתיבי אבחון (`/debug/*`)** - נבנו במקור לגישת-פיתוח פנימית, אין להם תפקיד כאן.
- **בלי תוסף-דפדפן** - אתר רגיל במקום Manifest V3 Chrome extension.
- **קוד-כניסה נפרד** - לא הערך האמיתי של המערכת המקורית.
- מסד-הנתונים (D1) והלוגיקה הפיננסית (חוב/זיכוי, מדיניות-תשלום, תחזית עומס) **זהים לגמרי** למערכת האמיתית - זה בדיוק אותו backend, רק עם מקור-נתונים אחר.
